# Roid — design notes

## Elevator pitch

**Roid** is a minimalist web game about farming on a procedurally generated voxel asteroid. The player works with a rotatable 3D rock that will later support modular gameplay (resources, planting, etc.). This document stays short and reflects what is implemented *now*.

## Current milestone

- Vite + TypeScript app with a full-viewport WebGL canvas.
- **Tips** dialog on first load (dismiss OK/scrim persists `roid:gameStartTipsDismissed`); **Settings (F10)** → **Tips** reopens the same copy.
- Procedural voxel asteroid (noise-warped sphere) on a fixed odd grid (`33³` cells in code). **`deriveAsteroidProfile(seed)`** combines three layers (deterministic): **spectral class** (C/S/M/D/V/E/X, renormalized main-belt–ish weights), **physical regime** (eleven: rubble/monolith/shattered/gardened/fresh/heated/volatile-rich plus contact-binary, collisional-family, outer-belt-cold), and **continuous dials** (shape: radius + noise; composition jitter; lithology hash scale; surface alteration; metal segregation). **Shape** from dials is then **modulated** (bounded multipliers) by spectral × regime before voxel generation. **Lithology** uses radial depth plus a small **seed-fixed axis** term so bands are not perfectly spherical; **`applyImpactCraters`** then carves seeded spherical bowl cavities (surface anchor via max radial dot, regime- and dial-tuned count/radius, min voxel floor; **Debug → balance**: impact radius **min/max (voxels)** define per-crater uniform random size (deterministic), then **radius multiplier** (0 = off); **min/max impact count** clamps totals; applies on Regenerate) before enrich; lithology + bulk bias feed enrich + composition. **`discoveryDensityScale(profile)`** scales **Debug → balance** discovery site density (e.g. volatile-rich / outer-cold slightly higher; monolith slightly lower; M/X nudged up; C nudged down). HUD **first line** is a short fingerprint (class · regime · r= · seed hex tail).
- **Spectral → root bias** (multiplicative on lithology templates; hand-tuned game analog, not a spectroscopic fit). Regime adds a second layer (e.g. outer-belt-cold → ices/hydrates; gardened → regolith).

| Class | Lean |
|--------|------|
| C | volatiles, carbonaceous, regolith; metals down |
| S | silicates, sulfides |
| M | metals, refractories; volatiles down |
| D | volatiles, carbonaceous, ices; metals down |
| V | silicates, oxides, metals |
| E | silicates, sulfides |
| X | metals, sulfides, refractories |
- **Resource tree:** twelve root commodities are mined and spent on structures; each root has a density band and **depth-scan susceptibility**. **Hubs** pull unrefined stock from the local voxel network into **global root tallies**; **refineries** spend roots from tallies and output **second-order (child) resources** (per-root recipes). **Refinery tool → Recipes** sets one **global active recipe** (which root is consumed); eleven roots are available from the start; **halides** unlocks at **computronium research tier 6** (same hidden / gibberish / unlocked pattern as lasers while researching). **Reactor, hub, and first refinery** convert costs use **roots only** (mineable before any refinery runs). **Replicator** tool placement spends **roots** per click (scaled by Debug → balance **resource cost multiplier**); spread to neighbors from eating is free. **Mature replicator → reactor / battery / hub / refinery / computronium** charges cost **immediately**, then the voxel **finishes transforming after a delay** (Debug → balance **Replicator → structure transform time**; live **Replicator transforms** list under Debug → Scan while the Debug section is open). Later **structure, unlock, satellite deploy, and explosive-arm costs** often mix roots and **refined children** (themed per tool tier; tool/HUD cost lines list all parts). HUD shows fingerprint, then non-zero roots, then non-zero refined children; **resource abbreviations on those two lines are color-coded** by **usual origin depth** (yield-weighted regolith / silicate / metal-rich bands; refined inherit parent root). Top-left **matter HUD** (`#matter-hud`) uses a **fixed capped width** (`min(480px, calc(100vw - 140px))`) so it stays clear of top-right controls; long lines wrap inside the panel; **default compact** typography/padding (Settings → **Compact resource HUD**, persisted `roid:matterHudCompact`, default on); **spacious** preset restores pre-compact padding and font size; a **corner minimize control** toggles the panel (persisted `roid:matterHudCollapsed`); minimized state **hides the blue panel** so only the **expand** control remains.
- Per-voxel **bulk composition** (normalized root fractions) drives mining/replicator yields and hub crediting from processed matter. After the usual lithology × spectral bias × jitter, a deterministic **rare-lode field** merges three graded morphologies — **pocket** (coarse regional noise), **vein** (seed-fixed sinusoidal slabs), **speckle** (sharp high-frequency hash) — with **spectral × regime** weights, then smoothstep + convex blend toward a **spectral-class rare template**; continuous strength `rareLodeStrength01` is kept on **processed matter** from parent rock.
- `InstancedMesh` rendering with subtle per-voxel color variation; default (un-scanned) rock tints blend lithology with a **bulk composition** hue (circular HSL blend over 12 roots, display-only fraction jitter, acid-or-gray saturation envelope + dominant-root emphasis; tunable under Debug → scan visualization).
- Orbit controls: drag to rotate the view, scroll to zoom (damped).
- Black void backdrop with sparse square-pixel stars in world space; star shell rotation uses the **same angular speed** (°/s) as the key light azimuth when **Rotate sun azimuth** is on (see Debug → Key light). Parallax when orbiting the rock.
- Norton Commander–style overlay UI (HUD, settings panels; tools are loose buttons only): VGA blue panels, cyan borders, DOS font [`public/fonts/perfect-dos-vga-437-win.ttf`](public/fonts/perfect-dos-vga-437-win.ttf). Bottom **tools dock** is a **border-only** frame (no panel fill behind the row); tool and sat buttons are **compact**, **fixed-size** cards; **build costs** are not inline—use **hover `title`** (tooltip) and a **status line** above the tool row for the **selected** tool (**short blurb** + **requirements**, or blurb only for Pick / Inspect). **+sat**, launch/decommission, **Recipes**, and **Killswitch** (Replicator tool) sit **to the right** of the main tool row (not below). **Satellite deploy** is contextual (same idea as Refinery → Recipes): the **+ Mining / Dig / Scan / Cleanup sat** button for a tier appears only when the **matching** laser tool (F7–F9) or **Cleanup (F13)** is selected and that tier is unlocked. Then two-step: pick **+sat**, read the **context line** (`have/need`), then **Launch satellite** or **Decommission** (modal confirm; if more than one of that type is deployed, the dialog asks **how many** to remove—same fleet rules as the orbit-marker modal). Settings → **Color scheme** switches the UI chrome between **Blue** (current VGA-like default), **Gray** (grayscale panels with orange/yellow/deep-blue accents), and **Orange** (Martian red/orange panels with ivory text and deep blue accents); the choice is persisted with other client settings.
- Tools: pick; **Inspect** (Ins, no cost): click a voxel → matter HUD readout (grid position, kind, HP or structure state); click an **orbit satellite marker** → modal with type, counts, orbit radius, and **Decommission** (confirm step); readout **clears when another tool is selected**; **refined preview / bulk density** only when that voxel has **surface scan tint** or **depth reveal progress** (same rules as scanner / depth sim). **replicator** (resource cost per placement); **Seed** (F15) is the early programming tool for replicators: click **Assembly** in the tools dock to open the Seed modal, pick a Seed type, and configure a short **slot program**: each slot chooses a recipe (or **Pause** / **Die**) plus a per-slot duration slider (seconds). The total Seed **lifetime** is the sum of slot durations, clamped to per-type min/max; slots marked **Pause** do no work for their duration, slots marked **Die** end the program when reached. The left side of the modal is a small **Seed inventory**: saved presets with name + config; you can **select**, **save/update**, or **delete** presets; the current selection (including ad-hoc edits) becomes the **active Seed** for planting and is persisted in `localStorage` (`roid:seedInventory`). The Seed lineup now grows over computronium tiers: **Basic Seed** stays simple and short with a couple of slots; mid-game adds **Efficient**, **Longlife**, and **Burst** Seeds with more slots and longer lifetimes; late-game unlocks **Scheduler**, **Macro**, and **Expert** Seeds with deep stacks (up to 10 slots) for long, pause-heavy choreographies that often end in a deliberate **Die**. Clicking voxels with the Seed tool plants the **currently active Seed**: on bare rock it spends the normal Replicator placement cost, starts a new replicator eating that rock, and binds the Seed program; while the Seed lifetime runs, **recipe slots** generate extra stored resources each tick, **Pause** slots just consume time, and hitting a **Die** slot terminates the Seed. Seed lifetime is chosen per Seed stack via slots, not hardwired by Seed type; each Seed type only provides the allowed range and max stack depth. Seed recipe slots only offer recipes that are both **allowed for that Seed type** and **unlocked by computronium research**; at game start the menu shows only a small set of **progression-critical** root recipes, with additional root families unlocking as tiers advance (plus any special discovery-based grants). **reactor** (F3) is **hidden** until at least one **mature replicator** voxel exists; - Scanner: click to spend energy on a **cube neighborhood** around the hit voxel (default half-extent **1** → 3×3×3); **Debug → balance** sets integer radius 0–4 and scales energy with volume. HUD **Scan →** line: **economic** refined-material preview (normalized roots → one-hop recipe blend → sharpened child weights; no display-only jitter).
- Explosive charge: click **any voxel** to arm; it **blinks** until detonation (~2.6 s), then destroys voxels in a **cube** around that cell; **Debug → balance** sets integer blast half-extent **0–4** (same neighborhood shape as scanner; no resource cost). Arms on **pointerdown**; orbit is paused until release **only when a fuse is armed** (background / miss still orbits). **Surface scan** stores a persistent per-voxel tint: same recipe pipeline as the HUD, plus **display-only** extras for discriminability — **scan-only** normalized-root micro-jitter, softer child sharpen than the HUD line, stronger child-weight and position perturbation, boosted HSL; sector HSL anchors per root family. At/above the Debug → balance **rare-lode strength floor**, surface and depth draw **only** the blue→red density heatmap (same gain as Debug **Show all lodes**), not composition-tint mixes; below the floor, rare-lode pockets still **lerp** toward the heatmap with **surface scan lode heatmap blend** / rare-lode lerp boost (surface) and depth **heatmap blend / saturation** (depth). Tint clears when the voxel is transformed. **Depth** overlay below the floor caches composition tint (`compositionToDepthScanColor` → `depthTintRgb`), then **lerps** toward the heatmap by graded density `rareLodeStrength01 × depthRevealProgress`; stronger **hint lerp** and **unrevealed brightness** than early versions; when **both** surface scan tint and depth overlay apply on sub-floor cells, depth shading is **slightly** lifted so saturation isn’t crushed before the scan lerp. **Depth scan** passive progress on rock / processed matter (Manhattan falloff), scaled by **composite depth-scan susceptibility** (`depthRevealSusceptibilityFloor` in Debug → balance). Depth overlay **see-through** uses **real** material alpha on solid + eating rock (`depthOverlayRockOpacity`; `depthWrite` off); instance RGB also **lerps toward void** using effective opacity from reveal progress, durability, and low-susceptibility boost (`depthOverlaySusceptibilityOpacityBoost`), with **full instance opacity** when reveal progress is high enough or **`rareLodeStrength01` is at/above the rare-lode strength floor** (Debug → balance). When graded density `min(1, rareLodeStrength01 × revealProgress)` is **≥** **depth overlay lode full opacity min density** (warm band vs cool/green), solid + eating use a **per-instance alpha multiplier** in the fragment shader so those voxels reach **full opacity** against the global rock transparency. Structures stay opaque. While Depth is on, solid + eating rock **instances are sorted back-to-front** each frame (camera-space distance) for correct transparency blending; **renderOrder** between the two rock layers follows which layer has the globally nearer voxel. **Overlays** menu toggles **Surface scan** and **Depth** without losing tints or progress.
- Dig laser sustain (audio): bandpass center is debug-tuned in a **low range** (**80 Hz–2.6 kHz**; default ~**1.28 kHz**).
- Ambient asteroid music: up to 12 voices (sine carriers for the general pad plus a dedicated **mid-range reese** voice built from a pair of detuned saws); pitch classes snap to a **scale clamp** (Major, Minor, Iwato, Hirajoshi, Major Pentatonic, or Locrian — Debug **Note pitch — scale clamp**) relative to the procedural root from the asteroid seed. **Scale cycle** (Debug, on by default): tonic pitch class advances along the **circle of fifths** or **fourths** on a timer (default **180 s**), with optional **interval jitter** (±seconds, seed-deterministic); carriers retarget via the existing pitch glide. Debug **Voices — pitch spread** scales deterministic inter-voice semitone spread and macro note-jitter depth (not macro transpose). Carriers **glide** in Hz toward that snapped pitch (exponential smoothing; Debug **base** seconds + **per-voice jitter** seconds, or 0+0 for instant steps); the reese has its **own** slide time (Debug **Mid reese — pitch slide time**) that does **not** depend on the global voice glide sliders, but still snaps to the same scale/scale cycle. Optional Debug **per-voice pitch-tracking bandpass** (checkbox) with **center offset in semitones** and macro **Q** vs the smoothed fundamental (same glide as the carrier). How many voices are active follows structure voxels + **all** satellite types (smoothed count; tunable in Debug). Optional **average voice lifetime** (0 = fixed first-N voices by index) turns on **stochastic churn** so which slots are on fluctuates over time even when activity is flat; optional **lifetime jitter** (0–1) spreads each slot’s mean lifetime around that average (seed-deterministic; affects death rate only). Each voice **fades in/out** on its own schedule; Debug sets **one** fade-in and fade-out **duration** (seconds) for all voices. Chorus + per-voice carrier and LFOs get **deterministic random initial phases** from the seed (so the same asteroid doesn’t stack every sine at phase 0). **Dual** sub-audio tremolo layers (slow additive + mid-rate subtractive at VCA), each with **LFO-on-LFO** rate wobble; **very slow** pan LFO per voice into **`StereoPannerNode`** before the bus. **Voice timbre macros** (Debug): one set of **center** controls applies to all 12 voices via **`applyVoiceMacrosToVoices`** (small deterministic per-voice spread); macro **note** offset + time jitter fold to **scale-snapped** semitone deltas for the current asteroid key (per-voice triad layout); note offset jitter **rate** has a wide log-Hz range plus optional **note jitter rate jitter** (depth + slow log Hz + sine/step, independent of LFO rate jitter); optional **music phrases** (macro: log **phrase rate** down to very sparse starts, optional **phrase rate jitter** depth + slow rate + sine/step, average length, depth) apply occasional spikes that boost note jitter depth and rate for short runs; optional **time-varying** LFO/pan **rate** jitter (**sine** or **step**; log-Hz rate sliders biased slow); shared **`voiceMacroJitterTimeSec`** clock in the music tick (resets on new asteroid seed); optional collapsed **per-voice overrides**. **Music bus** (Debug): chorus → pre waveshaper drive → lowpass (**log Hz slider**, 80–20k, travel biased low) with optional **very slow bus-only** cutoff LFO + **drifting LFO speed** (separate rates/phases from voice tremolo) → parallel convolver reverb (**wet/dry mix** + **wet trim**; full mix = no dry); **wet-only** post waveshaper after reverb; music debug sliders call **`applyDebugNow`** so bus params update immediately. **Music post chain** (Debug, music only): 3-band EQ + high-pass (default **40 Hz**, log slider 20–200 Hz) after dry/wet buses, then `destination`; persists `roid:audioMasterDebug` (separate from `asteroidMusicDebug`). Debug tunables for voices/bus persist in `localStorage` and optionally auto-save to [`asteroidMusicDebug.persisted.json`](src/game/asteroidMusicDebug.persisted.json) in dev (same pattern as game balance). Music volume in Settings (`localStorage`). Shared `AudioContext` with SFX.
- **Discoveries:** deterministic **discovery sites** per voxel (`isDiscoverySite` from asteroid seed + position; **site density** in Debug → **Discovery** is a **nominal fraction of voxels**, then scaled by **`discoveryDensityScale(profile)`** which adds spectral/regime tweaks plus a **per-asteroid random multiplier** in a bounded band. Even at a fixed slider value, different asteroids land slightly sparser or richer in sites. **Surface scan** and/or **Depth** overlay shows **opaque unlit bright red** only on voxels **already scanned by the scanner satellite** (persistent tint) that are still-eligible sites — not the whole rock at once. Scanning does **not** roll discoveries. A **claim** (pick, replicator rock HP, laser → processed matter, hub PM unit, depth scanner placement) consumes the site once and rolls a **discovery offer** from archetype weights (or **False Signal** if all weights are zero). By default **Settings → Resolve discoveries immediately** is **off**: new finds queue as **pulsing ◆ icons** in the viewport, **anchored beside the find site** with a **line** to that voxel, until clicked (then the modal opens; OK applies). Off-screen finds park **top-right** with the line hidden. Queued chips stay visible when the **resource HUD is minimized** (tools dock collapse is separate). When **on**, the modal opens immediately with the same anchoring. The modal shows a small deterministic **ASCII doodle** (not copied to the lore log) plus a **compact resource summary line** (e.g. `Resources: +3 Reg, -1 Fe`) whenever the outcome affects tallies. Outcomes: windfall/drain, lore (`roid:discoveriesLog`), or **research bypass** (tiers 1–4) via `applyResearchTierGrant` / unlock-point sync. The **depth scanner tool** is selectable while **tier-4 computronium research is in progress** (scanner tier done, ≥1 active computronium, points below tier 4), not only after depth unlock.
- **SFX bus:** one global parallel dry + convolver reverb for all tool feedback in [`clickFeedback.ts`](src/game/clickFeedback.ts) (lasers, dig sustain, mining thuds, scanner/hub/refinery/replicator sounds including replicator feeding); separate from the music convolver; IR from [`reverbImpulse.ts`](src/game/reverbImpulse.ts). Output goes to `AudioContext.destination` (no shared EQ/HPF with music). Debug → **SFX reverb (global)** sliders (wet send / wet out / impulse length / decay rate) persist with [`gameBalance`](src/game/gameBalance.ts).
- Key directional light: **azimuth and elevation** default from **random** on first run (no saved prefs); then persist in `localStorage` and [`settingsClient.persisted.json`](src/game/settingsClient.persisted.json) (dev auto-save / Save). **Regenerate** keeps saved light angles and key-light debug **°/s** (no re-roll); **intensity** still gets a small **random multiplier** (~0.9–1.1× baseline) per new asteroid. **Azimuth** slowly orbits when **Rotate sun azimuth** is on (default); slider **−120…+120°/s**; optional **direction helper** in **Debug → Key light**; that speed also drives **background star rotation**. Fill light stays fixed.
- Asteroid **root orientation** (Euler) is **random** on load and Regenerate so the rock does not always present the same face.

## Persistence

| Mechanism | What |
|-----------|------|

| Debug preset JSON | [`public/roid-debug-preset.json`](public/roid-debug-preset.json) is **auto-loaded at startup in published builds** (not in dev) via [`autoLoadDebugPreset.ts`](src/game/autoLoadDebugPreset.ts) before game state initialization, applying `localStorage` keys early so browser-only debug/settings travel with the release. Settings → **Download / Copy / Import debug preset** exports or applies the allowed `localStorage` keys (versioned `{ v: 1, entries }`); manual import **reloads** the page and takes precedence. ## Tech stack

- **Build:** Vite 8.x, TypeScript 5.9.x (`package.json` is source of truth for exact versions). **Tests:** Vitest (`npm test`) — deterministic checks for asteroid profile / voxel generation.
- **Rendering:** three.js (ES modules), `WebGLRenderer`, `MeshStandardMaterial`, `InstancedMesh`.
- **Types:** `@types/three` (dev); the installed three.js build does not ship its own `.d.ts` entry.
- **Camera:** `PerspectiveCamera` + `OrbitControls` (`three/addons/controls/OrbitControls.js`).
