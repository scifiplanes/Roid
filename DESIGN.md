# Roid — design notes

## Elevator pitch

**Roid** is a minimalist web game about farming on a procedurally generated voxel asteroid. The player works with a rotatable 3D rock that will later support modular gameplay (resources, planting, etc.). This document stays short and reflects what is implemented *now*.

## Current milestone

- Vite + TypeScript app with a full-viewport WebGL canvas.
- Procedural voxel asteroid (noise-warped sphere) on a fixed odd grid (`33³` cells in code). **`deriveAsteroidProfile(seed)`** combines three layers (deterministic): **spectral class** (C/S/M/D/V/E/X, renormalized main-belt–ish weights), **physical regime** (eleven: rubble/monolith/shattered/gardened/fresh/heated/volatile-rich plus contact-binary, collisional-family, outer-belt-cold), and **continuous dials** (shape: radius + noise; composition jitter; lithology hash scale; surface alteration; metal segregation). Shape params feed generation; **`applyImpactCraters`** then carves seeded spherical bowl cavities (surface anchor via max radial dot, regime- and dial-tuned count/radius, min voxel floor; **Debug → balance**: impact radius **min/max (voxels)** define per-crater uniform random size (deterministic), then **radius multiplier** (0 = off); **min/max impact count** clamps totals; applies on Regenerate) before enrich; lithology + bulk bias feed enrich + composition. HUD **first line** is a short fingerprint (class · regime · r= · seed hex tail).
- **Resource tree:** twelve root commodities are mined and spent on structures; each root has a density band and **depth-scan susceptibility**. **Hubs** pull unrefined stock from the local voxel network into **global root tallies**; **refineries** spend roots from tallies and output **second-order (child) resources** (per-root recipes). **Reactor, hub, and first refinery** convert costs use **roots only** (mineable before any refinery runs). **Replicator** tool placement spends **roots** per click (scaled by Debug → balance **resource cost multiplier**); spread to neighbors from eating is free. Later **structure, unlock, satellite deploy, and explosive-arm costs** often mix roots and **refined children** (themed per tool tier; tool/HUD cost lines list all parts). HUD shows fingerprint, then non-zero roots, then non-zero refined children; **resource abbreviations on those two lines are color-coded** by **usual origin depth** (yield-weighted regolith / silicate / metal-rich bands; refined inherit parent root). Top-left **matter HUD** (`#matter-hud`) uses a **fixed capped width** (`min(480px, calc(100vw - 140px))`) so it stays clear of top-right controls; long lines wrap inside the panel; **default compact** typography/padding (Settings → **Compact resource HUD**, persisted `roid:matterHudCompact`, default on); **spacious** preset restores pre-compact padding and font size; a **corner minimize control** toggles the panel (persisted `roid:matterHudCollapsed`); minimized state **hides the blue panel** so only the **expand** control remains.
- Per-voxel **bulk composition** (normalized root fractions) drives mining/replicator yields and hub crediting from processed matter.
- `InstancedMesh` rendering with subtle per-voxel color variation.
- Orbit controls: drag to rotate the view, scroll to zoom (damped).
- Black void backdrop with sparse square-pixel stars in world space; star shell rotation uses the **same angular speed** (°/s) as the key light azimuth when **Rotate sun azimuth** is on (see Debug → Key light). Parallax when orbiting the rock.
- Norton Commander–style overlay UI (HUD, settings panels; tools are loose buttons only): VGA blue panels, cyan borders, DOS font [`public/fonts/perfect-dos-vga-437-win.ttf`](public/fonts/perfect-dos-vga-437-win.ttf). Bottom **tools dock** is a **border-only** frame (no panel fill behind the row); tool and sat buttons are **compact**, **fixed-size** cards; **build costs** are not inline—use **hover `title`** (tooltip) and a **status line** above the tool row for the **selected** tool (**short blurb** + **requirements**, or blurb only for Pick / Inspect). **Satellite deploy** is two-step: pick a **+sat** type, read the **context line** (`have/need`), then **Launch satellite** or **Decommission** (modal confirm; if more than one of that type is deployed, the dialog asks **how many** to remove—same fleet rules as the orbit-marker modal).
- Tools: pick; **Inspect** (Ins, no cost): click a voxel → matter HUD readout (grid position, kind, HP or structure state); click an **orbit satellite marker** → modal with type, counts, orbit radius, and **Decommission** (confirm step); readout **clears when another tool is selected**; **refined preview / bulk density** only when that voxel has **surface scan tint** or **depth reveal progress** (same rules as scanner / depth sim). **replicator** (resource cost per placement); **reactor** (F3) is **hidden** until at least one **mature replicator** voxel exists; **hub** (F5) after **reactor**; **refinery** (F6) after **hub**; **battery** (F4) and **computronium** (F12) after **refinery** (resource cost: convert mature replicator → computronium voxel; active cells drain energy and accrue unlock progress faster with more active cells), mining laser, excavating laser, **scanner satellite** (unlock the three laser tools **only** via computronium progress or Debug → **Unlock all tools**; then deploy extra sats; energy per scan). **Laser tool buttons (F7–F9)** and **explosive charge (F10)** stay **hidden** until that tier is actively being researched (≥1 **on** computronium cell for lasers; F10 unlocks at **computronium research tier 1**, same milestone as mining laser); while unlocking, the button appears with **randomized gibberish** on the hotkey and label on a **slow interval** (~2s), with **fixed-width** lines so the card does not resize, then real labels once unlocked. **explosive charge** (any voxel → timed fuse → blast; per-arm resource + energy cost), **depth scanner** (fourth computronium research tier after scanner satellite — same unlock/Debug cheat as lasers; resource cost: convert unconverted rock or processed matter → depth scanner voxel). **Depth scan (F11)** uses the same hidden / gibberish / unlocked pattern as lasers while tier 4 is in progress. **Fifth computronium tier** unlocks **Dross Collector** satellites (`+ Dross sat`): deploy extra collectors to speed conversion. **Dross (F13)** in the tool row mirrors tier-5 hidden / gibberish / unlocked (informational; no voxel action). **Dross:** removing voxels (pick, dig laser, explosive blast) spawns small cubic **dross** particles near the cavity (instanced, parented to the asteroid); **replicators** add scrap **probabilistically** per rock HP (chance + mass in Debug → balance). Particle colors use **bulk rock hint** (scan viz). **`FogExp2`** scales with **total dross mass**; fog color is **light sRGB** by default so the shader **mix** toward fog reads as **bright haze** (lighten-like) rather than darkening. **Debug → balance**: density per mass, cap, and **R/G/B** (`drossFogDensityPerMass` / `drossFogDensityMax` / `drossFogColorR|G|B`; density 0 = off). No dross → no fog (background returns to black). While fog is on, **`scene.background`** lerps void → fog tint by **`density / cap`** (same ramp as dross strength), since the clear isn’t fogged in the shader. The **starfield** uses the same fog as the asteroid. Collectors convert dross mass into **root resources** using cluster bulk composition (rates in Debug → balance). Click computronium with the computronium tool to toggle off (no drain / no unlock contribution). Active computronium: emissive violet; off: dull. Satellite dots in four colors for mining / dig / scanner / dross collector.
- Scanner: click to spend energy on a **cube neighborhood** around the hit voxel (default half-extent **1** → 3×3×3); **Debug → balance** sets integer radius 0–4 and scales energy with volume. HUD **Scan →** line: **economic** refined-material preview (normalized roots → one-hop recipe blend → sharpened child weights; no display-only jitter).
- Explosive charge: click **any voxel** to arm; it **blinks** until detonation (~2.6 s), then destroys voxels in a **cube** around that cell; **Debug → balance** sets integer blast half-extent **0–4** (same neighborhood shape as scanner; no resource cost). Arms on **pointerdown**; orbit is paused until release **only when a fuse is armed** (background / miss still orbits). **Surface scan** stores a persistent per-voxel tint: same recipe pipeline as the HUD, plus **display-only** extras for discriminability — **scan-only** normalized-root micro-jitter, softer child sharpen than the HUD line, stronger child-weight and position perturbation, boosted HSL; sector HSL anchors per root family. Tint clears when the voxel is transformed. **Depth** overlay uses the same tint path (cached `depthTintRgb`); stronger **hint lerp** and **unrevealed brightness** than early versions; when **both** surface scan tint and depth overlay apply, depth shading is **slightly** lifted so saturation isn’t crushed before the scan lerp. **Depth scan** passive progress on rock / processed matter (Manhattan falloff), scaled by **composite depth-scan susceptibility** (`depthRevealSusceptibilityFloor` in Debug → balance). Depth overlay **see-through** uses **real** material alpha on solid + eating rock (`depthOverlayRockOpacity`; `depthWrite` off); instance RGB also **lerps toward void** using effective opacity from reveal progress, durability, and low-susceptibility boost (`depthOverlaySusceptibilityOpacityBoost`). Structures stay opaque. While Depth is on, solid + eating rock **instances are sorted back-to-front** each frame (camera-space distance) for correct transparency blending; **renderOrder** between the two rock layers follows which layer has the globally nearer voxel. **Overlays** menu toggles **Surface scan** and **Depth** without losing tints or progress.
- Dig laser sustain (audio): bandpass center is debug-tuned in a **low range** (**80 Hz–2.6 kHz**; default ~**1.28 kHz**).
- Ambient asteroid music: up to 12 sine voices in a major key from the asteroid seed; how many voices are active follows structure voxels + **all** satellite types (smoothed count; tunable in Debug). Each voice **fades in/out** on its own schedule; Debug sets **one** fade-in and fade-out **duration** (seconds) for all voices. Chorus + per-voice carrier and LFOs get **deterministic random initial phases** from the seed (so the same asteroid doesn’t stack every sine at phase 0). **Dual** sub-audio tremolo layers (slow + mid-rate), each with **LFO-on-LFO** rate wobble; **very slow** pan LFO per voice into **`StereoPannerNode`** before the bus. **Voice timbre macros** (Debug): one set of **center** controls applies to all 12 voices via **`applyVoiceMacrosToVoices`** (small deterministic per-voice spread); optional collapsed **per-voice overrides**. **Music bus** (Debug): chorus → pre waveshaper drive → lowpass (**log Hz slider**, 80–20k, travel biased low) with optional **very slow bus-only** cutoff LFO + **drifting LFO speed** (separate rates/phases from voice tremolo) → parallel convolver reverb (**wet/dry mix** + **wet trim**; full mix = no dry); **wet-only** post waveshaper after reverb; music debug sliders call **`applyDebugNow`** so bus params update immediately. **Music post chain** (Debug, music only): 3-band EQ + high-pass (default **40 Hz**, log slider 20–200 Hz) after dry/wet buses, then `destination`; persists `roid:audioMasterDebug` (separate from `asteroidMusicDebug`). Debug tunables for voices/bus persist in `localStorage` and optionally auto-save to [`asteroidMusicDebug.persisted.json`](src/game/asteroidMusicDebug.persisted.json) in dev (same pattern as game balance). Music volume in Settings (`localStorage`). Shared `AudioContext` with SFX.
- **Discoveries:** deterministic **discovery sites** per voxel (`isDiscoverySite` from asteroid seed + position; **site density** in Debug → balance, default low). **Surface scan** overlay shows **bright white** only on voxels **already scanned by the scanner satellite** (persistent tint) that are still-eligible sites — not the whole rock at once. Scanning does **not** roll discoveries. A **claim** (pick, replicator rock HP, laser → processed matter, hub PM unit, depth scanner placement) consumes the site once and may roll the modal (**modal chance** + archetype weights). By default **Settings → Resolve discoveries immediately** is **off**: new finds queue as **pulsing ◆ icons** above the resource HUD until clicked (then the modal opens; OK applies). Icons stay visible when the **resource HUD is minimized** (tools dock collapse is separate). When **on**, behavior matches the old immediate modal. The modal shows a small deterministic **ASCII doodle** (not copied to the lore log). Outcomes: windfall/drain, lore (`roid:discoveriesLog`), or **research bypass** (tiers 1–4) via `applyResearchTierGrant` / unlock-point sync. The **depth scanner tool** is selectable while **tier-4 computronium research is in progress** (scanner tier done, ≥1 active computronium, points below tier 4), not only after depth unlock.
- **SFX bus:** one global parallel dry + convolver reverb for all tool feedback in [`clickFeedback.ts`](src/game/clickFeedback.ts) (lasers, dig sustain, mining thuds, scanner/hub/refinery/replicator sounds including replicator feeding); separate from the music convolver; IR from [`reverbImpulse.ts`](src/game/reverbImpulse.ts). Output goes to `AudioContext.destination` (no shared EQ/HPF with music). Debug → **SFX reverb (global)** sliders (wet send / wet out / impulse length / decay rate) persist with [`gameBalance`](src/game/gameBalance.ts).
- Key directional light: **starting azimuth and elevation** are **random** on load and on **Regenerate** (same ranges as Settings sliders). **Azimuth** slowly orbits the asteroid (on by default). Default **°/s** magnitude is **random 0.00–10.00** with **random sign** per asteroid (new load + Regenerate); **intensity** gets a small **random multiplier** (~0.9–1.1× baseline) each time. Slider **−120…+120°/s** for overrides. Rotation on/off and optional **direction helper** are in **Debug → Key light**; that speed also drives **background star rotation** (when sun rotation is on). Fill light stays fixed.
- Asteroid **root orientation** (Euler) is **random** on load and Regenerate so the rock does not always present the same face.

## Controls

| Input | Action |
|--------|--------|
| First tap anywhere | Resumes the shared Web Audio context (needed on mobile before sound plays). |
| Left drag | Orbit camera around the asteroid |
| Scroll / pinch | Zoom in and out (clamped distance) |
| Bottom tool buttons | Select tool; cards are **fixed width**; **tooltip** (`title`) has full help. The **status line** above the tool row shows a **very short description** plus **resource requirements** for the selected tool (`label: short — costs`). **Pick** and **Inspect** (no placement cost) show **label: short** only. Resource lines show **inventory vs requirement** per commodity as `have/need` (and Charge adds `E current/need`); the satellite **context line** (below +sat when a type is selected) uses the same pattern. Tools with a resource cost are dimmed when unaffordable but **still selectable** so requirements stay visible (including when not met). **Inspect (Ins)** is always available (no cost): click the asteroid to append voxel info to the matter HUD (clears when you leave Inspect); click a **colored orbit marker** for satellite info and optional **Decommission** (confirmation dialog); detailed composition lines require **surface scan** or **depth reveal** on that voxel. **Reactor (F3)** is **omitted** until a mature replicator exists; **hub (F5)**, **refinery (F6)**, then **battery (F4)** and **computronium (F12)** gate in that order (see `structureToolPrereqs.ts`). **Laser tools (F7–F9)**, **explosive charge (F10)** (computronium research tier 1), and **Depth scan (F11)** are **omitted** until the matching computronium research tier is eligible; during unlock they show **gibberish** text; once unlocked, normal labels and behavior (or Debug **Unlock all tools**). **Dross (F13)** follows the same pattern for **tier 5** after depth scan (informational tool). **+ Mining / Dig / Scan / Dross sat** **select** a deploy type (does not spend); **Launch satellite** performs the deploy; **Decommission** opens a **modal** (Cancel / Decommission); with **multiple** satellites of that type, enter **how many** to remove (disabled when none deployed). Selecting a **main tool** clears sat selection. **+sat** buttons are **omitted** until that tier is unlocked (the sat row is hidden until at least one laser tier is unlocked). **+ Dross sat** follows the tier-5 deploy row (researching or unlocked; separate deploy cost). A **corner minimize control** on the tool dock collapses the tool + sat rows (persisted `roid:toolsBarCollapsed`). |
| Eye (overlays) | Toggle **Surface scan** and **Depth** overlays (tints / reveal progress stay stored when off). **Depth** is disabled until computronium tier-4 depth research is done **and** at least one depth scanner exists; placing the **first** scanner after that turns **Depth** on automatically (toggle off anytime). Overlay on/off states persist in `localStorage` (`roid:overlayVisualization`). With the menu open and at least one overlay on, a **Legend** lists refined-material hue families (anchor swatches) and short blurbs for the active overlays; the popover widens modestly and families lay out in a **responsive grid**; **Hide** / **Show** on the legend header collapses swatches (persisted `roid:overlayLegendCollapsed`). |
| F10 → Settings | Music volume (ambient); **Resolve discoveries immediately** (persisted `roid:discoveryAutoResolve`); **Compact resource HUD** (persisted `roid:matterHudCompact`, default on); regenerate asteroid; light elevation + manual azimuth when rotation off; Debug balance (incl. computronium drain / unlock tuning, **discovery site density + modal chance + archetype weights**), **Depth scan (debug)** (reveal rate / falloff / susceptibility floor / unrevealed + scanned voxel opacity / durability mix / susceptibility opacity boost — same fields as balance, `roid:gameBalance`), key-light rotation + direction helper, **scan visualization** (tint blend, HSL anchors, boost, emissive suppress; persisted in `roid:scanVisualizationDebug`), cheat row (**Unlock all tools**, add resources/energy, etc.), laser / dig / **replicator-feed** SFX, **global SFX reverb**, **music output** EQ + high-pass (music chain only), asteroid music (voice mapping + music bus FX); **Download / Copy / Import debug preset** (JSON of `localStorage` debug keys; import reloads). |

## Persistence

| Mechanism | What |
|-----------|------|
| Bundled JSON in repo | [`gameBalance.persisted.json`](src/game/gameBalance.persisted.json), [`asteroidMusicDebug.persisted.json`](src/game/asteroidMusicDebug.persisted.json) — merged at startup; in **dev**, Settings → Save / auto-save POSTs to Vite and overwrites those files. |

| Debug preset JSON | Settings → **Download / Copy / Import debug preset** exports or applies the allowed `localStorage` keys (versioned `{ v: 1, entries }`); import **reloads** the page. Commit or share the file to align another clone with your tuned browser-only debug. |

## Tech stack

- **Build:** Vite 8.x, TypeScript 5.9.x (`package.json` is source of truth for exact versions).
- **Rendering:** three.js (ES modules), `WebGLRenderer`, `MeshStandardMaterial`, `InstancedMesh`.
- **Types:** `@types/three` (dev); the installed three.js build does not ship its own `.d.ts` entry.
- **Camera:** `PerspectiveCamera` + `OrbitControls` (`three/addons/controls/OrbitControls.js`).
